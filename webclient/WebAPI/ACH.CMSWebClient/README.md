# 代码规范
1、完善日志，能精准定位异常位置
2、名称规范，描述同一个东西的名称要一致
3、页面接口类后缀加上【DTO】，用于和其余类区分
4、方法名称规范，如使用【Get/Convert/Update/Delete/Add/Query/Set/Save/Init】等
5、新增方法时先确定没有已经实现的类似的方法。如果有，优先扩展，避免重复开发

# 当前可优化点
1、接口返回信息冗余，且部分字段无用，可删除
2、接口类名称规范，见名思意，后缀统一为DTO
3、接口实现类优化，将相似功能封装合并，避免冗余
4、部分接口类实现过于复杂，是否可以重新设计，减少复杂的实现
5、数据下载部分功能代码优化
6、数据下载扩展螺栓波形筛选
